# PRD 품질 검토 결과 리포트

## 1. 기존 작성본 검토 결과 차트 (수정 전)

| 체크리스트 | 상태 | 평가 및 문제점 파악 |
| :--- | :---: | :--- |
| **1. Outcome-KPI 연계** | ⚠️ 보통 | 수치 및 임계치는 존재하나, 각 KPI를 어떤 도구나 시스템(측정 경로)을 통해 트래킹할지 명시되어 있지 않음. |
| **2. AC(Given-When-Then)** | ⚠️ 보통 | G-W-T 구조와 수치적 임계치는 잘 반영되었으나, **실패 케이스(예외 처리, 비정상 포맷 파일 등)**에 대한 검증 기준이 누락됨. |
| **3. 비기능 요구사항 (NFR)** | ✅ 양호 | 성능/신뢰성 수치(p95, SLA) 및 알림 기준(DataDog, Slack)이 구체적으로 매우 잘 명시됨. (수정 불필요) |
| **4. Differential Value** | ⚠️ 보통 | 대안 대비 "50배 단축", "99% 저렴" 등 수치는 있으나, 차별 가치로 명확히 분리하여 벤치마크 기준점(기존 체감 소요 시간 등)을 명세하지 않음. |
| **5. Proof** | ✅ 양호 | A/B 테스트 측정 도구와 목표 전환율 등 가설 검증 연결이 양호함. (수정 불필요) |

---

## 2. 챕터/파트별 수정 방향

*   **[섹션 1. 개요·목표 (성공 지표 파트)]:** 북극성 지표와 보조 KPI에 대해 정확한 데이터 수집 및 측정 경로(DataDog, Amplitude 등)를 추가하여 측정 가능성(Measurability)을 확보합니다.
*   **[섹션 3. 사용자 스토리와 수용 기준 (AC)]:** 정상 케이스(Happy Path) 외에 비정상 포맷 업로드, 결제 도중 이탈, 트래픽 초과 등의 실패 시나리오(Sad Path)에 대한 통제 수치를 AC로 추가하여 QA/엔지니어링 팀의 검증 가능성(Testability)을 극대화합니다.
*   **[섹션 4. 기능 요구사항 (차별 가치 보완)]:** 기능 설명 시, '기존 대안'과 '자사 솔루션'을 1:1로 수치 비교하여 차별 가치(Differential Value) 단락을 부각하도록 재작성합니다.

## 3. 수정 후 보완된 검토 결과 차트

| 체크리스트 | 상태 | 평가 및 문제점 파악 |
| :--- | :---: | :--- |
| **1. Outcome-KPI 연계** | ✅ 통과 | 측정 경로(Amplitude, DB Query 등)가 명시되어 즉각적인 지표 대시보드 구축이 가능해짐. |
| **2. AC(Given-When-Then)** | ✅ 통과 | 예외/에러 케이스 처리 시나리오가 추가되어 QA 엔지니어의 테스트 케이스 기반으로 직접 활용 가능해짐. |
| **3. 비기능 요구사항 (NFR)** | ✅ 통과 | (기존 유지) |
| **4. Differential Value** | ✅ 통과 | 기존 대안 vs 자사 솔루션 간의 시간/비용 수치 비교가 명확하게 보강됨. |
| **5. Proof** | ✅ 통과 | (기존 유지) |

## 4. 리라이트(Rewrite): 수정된 섹션

아래 내용은 검토 결과에 따라 측정 가능성 및 검증 가능성 강화를 위해 수정된 섹션(1, 3, 4)들입니다. 이외 섹션은 원본 PRD를 유지합니다.

---

### ## 1. 개요·목표
*(문제 정의 및 목표는 원문 유지, 성공 지표 부분만 수정 적용)*

**- 성공 지표(북극성/보조 KPI):**
  - **북극성 KPI (North Star KPI):** 유료 '데이터맵 ZIP 패키지' 다운로드 완료 건수
    - **통제/목표:** 기준선 0건 ➔ 목표 월 10,000건 달성
    - **측정 주기 및 경로:** 매주(Weekly) 집계 / PG 결제 완료 콜백 API 호출 수 및 DB상 `payment_cleared = true` 레코드 스캔 (Amplitude 대시보드에 이벤트 전송).
  - **보조 KPI 1:** 업로드된 비정형 문서 대비 폼 파싱 완료율
    - **통제/목표:** 95% 이상 방어
    - **측정 주기 및 경로:** 매일(Daily) 집계 / 파싱 파이프라인(DataDog APM) 상 `doc_id` 인입 건수 대비 정상적으로 `form_id`가 발급된 응답 비율 확인.
  - **보조 KPI 2:** 바이럴 폼 하단 '워터마크' 유입 대비 서비스 가입 전환율
    - **통제/목표:** 기준선 0% ➔ 목표 5% 이상 달성
    - **측정 주기 및 경로:** 주간(Weekly) 집계 / 워터마크 버튼에 달린 `utm_source=watermark` 파라미터 클릭 유입 수 대비 실제 회원가입 완료 수 (GA4 퍼널 분석).


### ## 3. 사용자 스토리와 수용 기준(AC, Acceptance Criteria)
*(기존 AC에 실패/예외 처리 등 검증을 위한 Sad Path 테스트 케이스 추가)*

**Story 1: 문서 무손실 파싱 (홍일반 / 최실무 공통)**
> **As a** 설문 작성자(기획자/일반인), **I want** 질문지가 적힌 워드/HWP 문서를 시스템에 업로드하여 즉시 설문 폼을 자동 생성하고 싶다, **so that** 문항을 복사-붙여넣기하는 지루한 입력 노동을 없앨 수 있다.
- **AC 1 (정상 처리):** Given 50문항 이내(텍스트 5MB 이하)의 HWP/Word/PDF 파일이 주어졌을 때, When [자동 폼 변환] 버튼을 클릭하면, Then **10초(10,000ms) 이내**에 파싱 및 모바일 웹 렌더링이 완료되어야 한다.
- **AC 2 (정확도 검증):** Given 변환된 폼을 검토할 때, When 원본 문서의 항목 수와 DB 스키마(`structure_schema`)를 연산 비교하면, Then 파싱 누락으로 인한 **데이터 손실률이 1% 미만**이어야 한다.
- **AC 3 (기능 검증):** Given 무료 사용자가 생성한 폼에 응답자가 접속할 때, When 화면이 로드되면, Then 하단 뷰포트에 **워터마크 배너가 100% 렌더링(노출)**되어야 한다.
- **AC 4 (예외 처리 - 🚨신규):** Given 암호가 걸려있거나 손상된 파일, 혹은 지원 범위 외의 확장자(.txt 등)를 업로드했을 때, When 파싱을 시도하면, Then **2초 이내에 명확한 실패 사유가 포함된 에러 모달(Error Code: 400)**을 띄우고 상태를 `FAILED`로 기록해야 한다.

**Story 2: 4종 산출물 ZIP 턴키 출하 (최실무 캐시카우 핵심)**
> **As a** 신사업 기획 실무자, **I want** 데이터 수집 종료 직후 응답 원본 엑셀뿐 아니라 "변수가이드, 코드북, 데이터맵"이 묶인 대행사급 4종 패키지를 다운로드하고 싶다, **so that** 응답 내용을 일일이 숫자와 코드로 변환하는 수작업 야근을 즉시 퇴근으로 바꿀 수 있다.
- **AC 1 (정상 연동):** Given 조사가 종료(수집 마감)되었을 때, When 대시보드에서 [보고용 데이터 패키지 다운로드] 버튼을 누르면, Then **3초 이내**에 PG사 결제 모듈 프레임이 오류 없이 팝업되어야 한다.
- **AC 2 (정확도 검증):** Given 다운로드된 데이터맵(Data Map) 파일에 대해, When 백엔드 데이터 검증을 수행했을 때, Then 전체 응답자 레코드 중 형식 불일치나 **결측치(Missing value) 처리 실패율이 0%**임이 보장되어야 한다.
- **AC 3 (예외 처리 - 🚨신규):** Given 결제 진행 중 유저가 창을 닫아 도중 이탈(Cancel) 하거나 잔액 부족으로 결제 실패 코드를 수신받았을 때, When 시스템이 이를 감지하면, Then DB의 `payment_cleared=false`를 유지하고 ZIP 파일의 **안전한 S3 다운로드 서명 URL 발급을 즉각 차단(403 Forbidden)**해야 한다.

**Story 3: 동적 쿼터/라우팅 인프라 (유팀장 VIP 핵심)**
> **As a** 리서치 에이전시 운영팀장, **I want** 엑셀 파일 업로드 기반 노코드로 동적 다중 쿼터(성별x연령x지역)와 외부 패널사 라우팅(Redirect)을 제어하고 싶다, **so that** 주말 모니터링 알바비와 조사별 외주 스크립트 개발비를 지출하지 않고 마진을 100% 수취할 수 있다.
- **AC 1 (정상 처리):** Given 패널 연동을 세팅할 때, When 상태별 포스트백 링크(성공/스크린아웃/쿼터풀)를 UI에서 입력하면, Then 해당 웹훅 및 **라우팅 실패로 인한 패널 이탈률이 0.1% 미만**으로 통제되어야 한다.
- **AC 2 (쿼터 검증):** Given 교차 쿼터 목표치(예: 20대 여성 100명)에 이미 도달했을 때, When 100번째 응답자 이후 조건 합치자가 진입하면, Then 트래픽 초과로 인한 **Over-quota 수용 오차율이 1% 이내**여야 하며 즉시 '할당초과(Quota Full)' URL로 리다이렉션 시켜야 한다.
- **AC 3 (예외 처리 - 🚨신규):** Given 동시 접속자가 1,000명을 초과하는 트래픽 스파이크 상황에서, When 쿼터 카운트 로직을 수행할 때, Then 연산 병목으로 인한 **DB 데드락(Deadlock)이 발생하지 않아야 하며**, 쿼터 연산 레이턴시가 1초를 초과할 경우 모니터링 시스템(DataDog)으로 **경고(Alert)를 발송**해야 한다.


### ## 4. 기능 요구사항(Functional)
*(기능 설명 시, 비교 대상과 차별 가치(Differential Value)를 수치화하여 명확한 평가 기준이 되도록 재작성)*

- **[Must] 비정형 문서 고정밀 파서 (AI Document Parser):**
  - **차별 가치 (Differential Value):** 기존 구글폼/네이버폼의 인간 수기 입력 소요 시간(50문항 기준 평균 30분) 대비, 파싱 모듈 소요 시간을 **10초로 선형 단축하여 작업 시간 효율을 18,000% 향상**시킴. (QA 진행 시 입력 시작부터 생성 완료까지의 체감 시간 벤치마킹 필수).
- **[Must] 데이터맵 / 코드북 컴파일러 (산출물 ZIP 추출기):**
  - **차별 가치 (Differential Value):** 기존 리서치 대행사 이용 시 소요되는 데이터 맵핑 대기 시간인 '최소 24시간'을 **'조사 종료 즉시(대기시간 5초 이내)'로 단축**. 용역 외주 비용 또한 거시적 수백만 원 대의 지출에서 통합 **'29,900원' 수준으로 설정해 99% 비용 절감 효과 창출**.
- **[Must] 다운로드 잠금 기반 결제 월(Paywall) UI:**
  - Aha-Moment(데이터맵 다운로드) 기반 강력한 록인(Lock-in) 장치 설계. 결제 미완료 시 출력 차단 기능 필수.
- **[Should] 노코드 동적 다중 쿼터 세팅 UI (B2B 에이전시 락인용):**
  - **차별 가치 (Differential Value):** 외부 스크립터 고용으로 발생하는 커스텀 쿼터 개발 리드타임(최소 2~3일)과 외주 비용(건당 최소 10만 원)을 **전면 클라우드 노코드 조작 체계(소요시간 5분, 외주비 0원)로 100% 내재화**.
- **[Won't] 화려한 에디터 템플릿 / 색상 및 폰트 변경툴:** 초기 사업 목표인 '분석 데이터 결과물 출하 자동화'에 부합하지 않으며, 구매 결정 요소와 무관하여 V1 MVP 개발 스펙에서 배제함.
