요청하신 `SRS_v0.1_Opus.md`와 `SRS_v0.2_Flash.md` 간의 주요 변경 사항을 3가지 관점으로 검토하여 정리한 보고서입니다.

전체적으로 **v0.2(Flash)**는 1인 개발자가 **Zero-Cost(무료 티어)**로 빠르게 배포할 수 있는 **'Indie MVP' 형상**으로 대폭 최적화되었습니다.

---

### 1. 기술 스택의 명확성 (Technology Stack Clarity)

v0.1은 엔터프라이즈급 구성인 반면, v0.2는 실제 구현 가능한 구체적인 **Serverless/BaaS 솔루션**을 명시했습니다.

| 항목 | v0.1 (Opus) | v0.2 (Flash) | 변경 요지 |
| :--- | :--- | :--- | :--- |
| **AI 엔진** | AI Document Parser (범용) | **Gemini 1.5 Flash API** | 구체적인 LLM 모델 명시 및 비용 효율성 확보 |
| **인프라/호스팅** | AWS (S3 등) / 범용 API 서버 | **Vercel** / **Supabase** | 1인 개발/배포에 최적화된 Serverless 환경 |
| **데이터베이스** | Database (범용) | **Supabase (PostgreSQL)** | BaaS를 통한 개발 공수 및 관리 비용 절감 |
| **캐시/제한** | Redis | **Upstash (Serverless Redis)** | 인프라 관리 부담 없는 서버리스 Redis 도입 |
| **모니터링** | DataDog, PagerDuty | **Slack Webhook** | 과도한 모니터링 도구를 단순한 실시간 알림으로 통합 |

---

### 2. MVP 목표의 조정 내용 (MVP Goal Adjustments)

복잡도가 높은 기능을 과감히 도려내고, 핵심 가치(문서 파싱 → 설문 생성)에 집중하도록 조정되었습니다.

| 항목 | v0.1 (Opus) | v0.2 (Flash) | 변경 요지 |
| :--- | :--- | :--- | :--- |
| **운영 비용** | 월 서버 예산 500 USD | **월 0원 (Free Tier 활용)** | 수익 발생 전까지 지출 제로화 달성 목표 |
| **대상 문서** | HWP, Word, PDF | **Word, PDF** (HWP 제외) | HWP 파싱 기술 리스크를 Phase 2로 연기하여 출시 우선 |
| **핵심 기능** | 쿼터 제어, 외부 패널 라우팅 포함 | **쿼터/라우팅 기능 제외** | 개발 난이도가 높은 B2B 대행사 기능 제외 |
| **보안 정책** | 24시간 내 삭제 스케줄러 | **Edge Cron Job 기반 일괄 삭제** | Vercel 기능을 활용한 구현 방식의 단순화 |
| **성능 목표** | p95 응답 300ms 이내 | **p95 응답 2,000ms 이내** | 서버리스 콜드 스타트를 고려하여 현실적 목표 수정 |

---

### 3. 기타 차이점 (Other Differences)

사용자 경험과 운영 환경의 현실적인 타협점들이 반영되었습니다.

| 항목 | v0.1 (Opus) | v0.2 (Flash) | 비고 |
| :--- | :--- | :--- | :--- |
| **이해관계자** | 유팀장(대행사), 보안보스 포함 | **1인 개발자(Owner)** 중심 | 의사결정 구조의 단순화 및 본인 직접 개발 전제 |
| **가용성(SLA)** | 월 99.9% 이상 보장 | **Best Effort** 원칙 | 무료 티어 서비스(Vercel 등)의 가용성에 의존 |
| **파싱 완료율** | 95% 이상 목표 | **90% 이상** 목표 | 초기 엔진의 안정성을 고려한 목표치 현실화 |
| **데이터 정밀도** | 손실률 1% 미만 | **인식률 90% 이상** (핵심 텍스트) | 정밀한 구조 분석보다 핵심 내용 보존에 초점 |
| **확장성** | 동시 접속자 1,000명 이상 | **동시 접속자 50~100명 수준** | Supabase 무료 티어의 연결 한계를 고려한 현실적 설계 |

---

**검토 결론:**
`v0.2_Flash`는 `v0.1_Opus`의 풍부한 기능을 담백하게 걷어내고, **"실제로 내일 당장 개발을 시작하여 0원으로 런칭할 수 있는 문서"**로 진화했습니다. 기술 스택이 **Gemini/Vercel/Supabase**로 명확히 고정됨에 따라 개발 속도가 획기적으로 향상될 것으로 판단됩니다.
